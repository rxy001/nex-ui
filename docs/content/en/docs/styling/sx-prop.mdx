# The sx Prop

The `sx` prop is a shorthand for defining custom styles with direct access to the theme.

The `sx` prop provides a superset of CSS. In addition to standard CSS properties and selectors, it includes Nex UI-specific properties and selectors. CSS property values can directly reference the predefined theme.
If the built-in custom properties, selectors, and theme options are not enough, Nex UI is fully extensible. Learn more on the [Customizing the Theme page](/docs/customization/theming).

## Usage

The following example shows how to use the `sx` prop. Note that `sx` keys are not limited to standard CSS properties, because they map to specific token types. This concept is explained in more detail below.

<FlexCenter justify='center'>
  <Box
    sx={{
      maxWidth: 390,
      borderRadius: 'xl',
      boxShadow: 'lg',
    }}
  >
    <Box sx={{ display: 'flex', gap: '3', alignItems: 'center', p: '3' }}>
      <Box as='img' src={logo.src} width={50} height={50} />
      <Box>
        <Box>Nex UI</Box>
        <Box
          sx={{
            fs: 'sm',
            color: 'rgb(113, 113, 122)',
          }}
        >
          nexui.dev
        </Box>
      </Box>
    </Box>
    <Box as='hr' sx={{ m: 0, border: 'none', bg: 'gray.200', height: 1 }} />
    <Box sx={{ p: '3' }}>Make beautiful websites with ease and efficiency.</Box>
    <Box as='hr' sx={{ m: 0, border: 'none', bg: 'gray.200', height: 1 }} />
    <Box
      sx={{
        p: '3',
        display: 'inline-block',
        textDecoration: 'none',
        color: 'blue.600',
      }}
      as='a'
      href='https://github.com/rxy001/nex-ui'
      target='_blank'
    >
      Visit source code on GitHub.
    </Box>
  </Box>
</FlexCenter>

## Theme-Aware Properties

With Scales, you can map CSS properties to `tokens` and `semanticTokens`, allowing each property to use values from the corresponding token type.

```tsx {11,12,13}
import { Box, defineTheme, NexUIProvider } from '@nex-ui/react'

const theme = defineTheme({
  tokens: {
    colors: {
      primary: 'blue',
    },
    sizes: {
      1: '10px',
    },
  },
  scales: {
    color: 'colors',
    width: 'sizes',
    height: 'sizes',
  },
})

export default function Example() {
  return (
    <NexUIProvider theme={theme}>
      <Box
        sx={{
          // Only when color: 'colors' is defined in scales can the color property use tokens.colors.primary.
          color: 'primary',
          // At this time, the value of the width property will be compiled into tokens.sizes.1.
          width: '1',
          // Only when the value is a string can it be potentially mapped to tokens. At this point, the value of width is 1px.
          height: 1,
        }}
      />
    </NexUIProvider>
  )
}
```

Note that a CSS property value can be mapped to tokens only when the value is a string.

In the above example, the actual value of `width` is `10px`, while the actual value of `height` is `1px`.

Read more on the [Scales page](/docs/customization/scales).

### Colors

Properties such as `color` and `backgroundColor` can accept paths from `tokens.colors` and `semanticTokens.colors`:

```tsx
<Box sx={{ color: 'blue.400' }} />
```

The `backgroundColor` property can also be accessed via the alias `bg`:

```tsx
<Box sx={{ bg: 'blue.400' }} />
```

Read more on the [Colors page](/docs/customization/colors).

### Borders

The `borderWidth` property can accept paths from `tokens.borders` and `semanticTokens.borders`:

```tsx
<Box sx={{ border: 'sm', borderColor: 'blue.500' }} />
```

## Dark Mode

All CSS properties in `sx` also support dark mode. When used with Nex UI's Color Mode system, you can apply dark-mode styles with `_dark`, light-mode styles with `_light`, and default styles with `_DEFAULT`.

The following example demonstrates how to define styles for components in different modes:

<FlexCenter>
  <Box
    sx={{
      width: '100%',
      borderRadius: 'md',
      p: '3',
      bg: {
        _dark: 'white',
        _light: '#4c4c54',
        _DEFAULT: '#4c4c54',
      },
      color: {
        _dark: '#4c4c54',
        _light: 'white',
        _DEFAULT: 'white',
      },
      textAlign: 'center',
    }}
  >
    Color Mode
  </Box>
</FlexCenter>


## `colorPalette` Prop

The `colorPalette` property creates a color placeholder for easier theming and customization.
To use `colorPalette`, your colors should follow a consistent naming convention. By default, Nex UI provides scales from 50 to 900 for each color, making `colorPalette` straightforward to use.

Suppose we need to create a button with themeable styles from scratch.

<FlexCenter>
  <Box
    sx={{
      colorPalette: 'blue',
      bg: {
        _DEFAULT: 'colorPalette.500',
        _hover: 'colorPalette.400',
        _active: 'colorPalette.600',
      },
      p: '5px 15px',
      color: '#fff',
      borderRadius: 'lg',
      textAlign: 'center',
    }}
  >
    Hello World
  </Box>
</FlexCenter>

## Token Reference Syntax

Normally, you can reference a token type in CSS only when the mapping between CSS properties and token types is defined in `scales`. Nex UI includes default `scales` settings:

```tsx
{
  color: 'colors',
  backgroundColor: 'colors',
  ...
}
```

Therefore, in the `sx` property, you can use:

```tsx
<Box
  sx={{
    color: 'gray.100',
    bg: 'blue.500',
  }}
>
  Box
</Box>
```

However, with shorthand properties such as `border` and `padding`, tokens cannot be referenced in the same way. To solve this, Nex UI provides token reference syntax,
which allows tokens to be used in shorthand properties:

```tsx
<Box
  sx={{
    border: '{borderWidths.sm} solid {colors.blue.500}',
  }}
>
  Box
</Box>
```

Moreover, you can also use `colorPalette` in the token reference syntax:

```tsx
<Box
  sx={{
    border: '{borderWidths.sm} solid {colors.colorPalette.500}',
    colorPalette: 'blue',
  }}
>
  Box
</Box>
```

In this case, `border` resolves to the values of `borderWidths.sm` and `colors.colorPalette.500`.

## Color Opacity Modifier

When a CSS property is mapped to the `colors` type in `scales`, you can use the `opacity` modifier to adjust transparency, whether you provide a raw color value or a color token.

This also works with token reference syntax. When referencing a color token, you can apply the `opacity` modifier to create a wider range of visual effects.

For example, `color: 'blue.500/50'` sets the transparency to 50%.

```tsx
<Box
  sx={{
    color: 'blue.500/50',
    bg: 'orange/50',
    border: '1px solid {colors.colorPalette.500/50}',
    borderTopColor: 'colorPalette.primary/90',
    colorPalette: 'red',
  }}
>
  Box
</Box>

/**
 * {
 *   color: color-mix(in srgb, var(--nui-colors-blue-500) 50%, transparent);
 *   background-color: color-mix(in srgb, orange 50%, transparent);
 *   border: 1px solid color-mix(in srgb, var(--nui-colors-red-500) 50%, transparent);
 *   border-top-color: color-mix(in srgb, var(--nui-colors-red-primary) 90%, transparent);
 * }
 **/
```

## Array Values

Array syntax is useful when you want later entries to partially override earlier styles:

```tsx
<Box
  sx={[
    {
      '&:hover': {
        color: 'red',
        backgroundColor: 'white',
      },
    },
    foo && {
      '&:hover': { backgroundColor: 'grey' },
    },
    bar && {
      '&:hover': { backgroundColor: 'yellow' },
    },
  ]}
/>
```

When you hover over this element, `color: red; backgroundColor: white;` is applied.

If `foo: true`, `color: red; backgroundColor: grey;` is applied on hover.

If `bar: true`, `color: red; backgroundColor: yellow;` is applied on hover regardless of `foo`, because higher array indexes take precedence.

## Aliases

You can define aliases for CSS properties through [Aliases](/docs/customization/aliases), thus simplifying CSS property names and supporting the combination of CSS properties.

```tsx
<Box sx={{ bg: 'blue.400', px: '2' }} />
```

## Selector

If you need to combine complex selectors, such as `'&:not(:disabled):not([data-disabled=true]):hover'`, you can define additional selectors via Selectors to support more advanced use cases.

Read more on the [Selectors page](/docs/customization/selectors).

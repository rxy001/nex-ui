# Tokens

Design tokens are used to manage design decisions for an application. It is a collection of properties used to describe any basic/atomic visual styles.

## Customizing Tokens

Tokens can be defined in the `theme.tokens` section of your theme configuration:

```tsx
import { defineTheme } from '@nex-ui/react'

const theme = defineTheme({
  tokens: {
    colors: {
      black: '#000',
      sky: {
        50: '#E6F1FE',
        100: '#CCE3FD',
        200: '#99C7FB',
        300: '#66AAF9',
        400: '#338EF7',
        500: '#006FEE',
        600: '#005BC4',
        700: '#004493',
        800: '#002E62',
        900: '#14204a',
      },
    },
    spaces: { 16: '64px', 17: '68px', 18: '72px' },
  },
})
```

Tokens are pieces of information associated with an intuitive and readable name, generally in the form of name/value pairs. Except for color tokens, they also support the above object format.

<Callout>
  It is recommended that you define some objective tokens in `tokens` that have
  no connection with business scenarios and brand concepts.
</Callout>

### TypeScript

Later, if you are using TypeScript, you can extend the custom tokens through [Module Augmentation](https://www.typescriptlang.org/docs/handbook/declaration-merging.html#module-augmentation).

```ts
declare module '@nex-ui/react' {
  interface TokensOverrides {
    colors:
      | 'black'
      | 'sky.50'
      | 'sky.100'
      | 'sky.200'
      | 'sky.300'
      | 'sky.400'
      | 'sky.500'
      | 'sky.600'
      | 'sky.700'
      | 'sky.800'
      | 'sky.900'
    spaces: '16' | '17' | '18'
  }
}
```

## Using Tokens

After being defined, you can use the custom tokens through the [`sx`](/docs/styling/sx-prop) property, [`defineRecipe`](/docs/styling/recipes) or [`defineSlotRecipe`](/docs/styling/slot-recipes).

```tsx
import { Box } from '@nex-ui/react'

export default function App() {
  return (
    <Box
      sx={{
        backgroundColor: 'sky.500',
        padding: '16',
      }}
    >
      Box
    </Box>
  )
}
```

<Callout>
  Nex UI has associated the mapping between `backgroundColor` and `colors`,
  `padding` and `spaces`. Therefore, they can be used directly. If you change
  `backgroundColor: 'sky.500'` to `outlineColor: 'sky.500'`, it will not work
  properly. Read more on the [Scales](/docs/customization/scales) page.
</Callout>

## Token Types

### Colors

Color tokens represent the visually presented colors of elements. Their value type is `string | object`.

```tsx
import { defineTheme } from '@nex-ui/react'

const theme = defineTheme({
  tokens: {
    colors: {
      red: {
        100: '#fff1f0',
      },
    },
  },
})
```

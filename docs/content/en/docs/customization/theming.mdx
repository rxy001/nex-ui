# Theming

Customize Nex UI with your own theme to change colors, typography, and more.

Themes let you apply a consistent tone to your app. They allow you to customize all design aspects of your project to meet the specific needs of your business or brand.

To promote greater consistency between apps, light and dark theme types are available to choose from. By default, components use the light theme type.

## Nex UI Provider

Nex UI ships with a built-in default theme. You can apply a custom theme through the `theme` prop on `NexUIProvider`.

You can also generate a theme with `defineTheme` and pass the result to `NexUIProvider.theme`.

```tsx
import { defineTheme, NexUIProvider } from '@nex-ui/react'

const theme = defineTheme({
  tokens: {
    black: '#000',
    sky: {
      50: '#f0f9ff',
      100: '#dff2fe',
      200: '#b8e6fe',
      300: '#74d4ff',
      400: '#00bcff',
      500: '#00a6f4',
      600: '#00a6f4',
      700: '#0069a8',
      800: '#00598a',
      900: '#024a71',
    },
  },
})

export default function App() {
  return (
    <NexUIProvider theme={theme}>
      <Button>button</Button>
    </NexUIProvider>
  )
}
```

## Configurable Theme Properties

Adjusting theme configuration is the most effective way to tailor Nex UI to your needs.
The following links cover all theme properties:

- [`scales`](/docs/customization/scales)
- [`tokens`](/docs/customization/tokens)
- [`semanticTokens`](/docs/customization/semantic-tokens)
- [`breakpoints`](/docs/customization/breakpoints)
- [`aliases`](/docs/customization/aliases)
- [`selectors`](/docs/customization/selectors)
- [`components`](/docs/customization/components)

You can add additional properties to the theme, which can then be used elsewhere. Check the above links for more information.

```tsx
import { defineTheme } from '@nex-ui/react'

const theme = defineTheme({
  tokens: {
    colors: {
      black: '#1a1e23',
      blue: {
        500: '#006FEE',
      },
    },
  },
  semanticTokens: {
    colors: {
      primary: '#338EF7',
      text: {
        heading: '#0f172b',
        body: '#314158',
      },
    },
  },
})
```

## Nested Themes

You can nest `NexUIProvider` to change the theme for specific parts of your UI.

<FlexCenter>
  <Checkbox defaultChecked color='red' />
  <Checkbox defaultChecked color='green' />
</FlexCenter>

```tsx
import { NexUIProvider, defineTheme } from '@nex-ui/react'

const outerTheme = defineTheme({
  components: {
    Checkbox: {
      defaultProps: {
        color: 'red',
      },
    },
  },
})

const innerTheme = defineTheme({
  components: {
    Checkbox: {
      defaultProps: {
        color: 'green',
      },
    },
  },
})

export default function App() {
  return (
    <NexUIProvider theme={outerTheme}>
      <Checkbox defaultChecked />
      <NexUIProvider theme={innerTheme}>
        <Checkbox defaultChecked />
      </NexUIProvider>
    </NexUIProvider>
  )
}
```

For nested `NexUIProvider` instances, only `theme.components` and `theme.primaryThemeColor` take effect.
Other theme properties only apply at the top-level `NexUIProvider`.

## Theme Color

You can configure the default component theme color with `theme.primaryThemeColor`.

<FlexCenter gap='5'>
  <Checkbox defaultChecked color='orange' />
  <Switch defaultChecked color='orange' />
  <Button color='orange'>Button</Button>
</FlexCenter>

```tsx
import { NexUIProvider, defineTheme } from '@nex-ui/react'
import { Button } from '@nex-ui/react'

const theme = defineTheme({
  primaryThemeColor: 'orange',
})

export default function App() {
  return (
    <NexUIProvider theme={theme}>
      <Checkbox defaultChecked />
      <Switch defaultChecked />
      <Button>Button</Button>
    </NexUIProvider>
  )
}
```

## API

### defineTheme

`defineTheme(options) => theme`

Generates a theme from the provided options. Pass the result to your theme provider.

### defineColorScheme

`defineColorScheme(options) => colorScheme`

Generates a color scheme from the provided options. Pass the result to your theme provider.
